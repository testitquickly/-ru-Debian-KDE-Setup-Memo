===== Что такое «bystro.linux» =====

< @main @about >

Меня зовут Алексей Лупàн, я «головний спiвпрацювальник по підготувальні тестувальників în Moldova» — https://testitquickly.com/about/ 
Интересуюсь сложными задачами по тест-дизайну, баритон-гитарой и книгами Роберта Хайнлайна.

За десятилетия возни с Debian у меня собрался txt-файл с рецептами и подсказками. Plain text идеален для copy\paste, но в txt всё плохо с иерархией большого массива текстов и перекрестными ссылками, нужна  локальная «википедия», в которой останутся сила и ясность plain text. Собрал всё в [[https://github.com/zim-desktop-wiki/zim-desktop-wiki|Zim]] — фантастише мультиплатформед-десктоп-текст-редактор wiki-pages на основе plain text файлов с markdown разметкой, и получилась инструкция для быстрой и последовательной настройки KDE и сопутствующего ПО сразу после установки Debian или Manjaro. 

Название «bystro.linux» придумалось само и ничего особенно не подразумевает.

	Ещё собрался https://github.com/testitquickly/bystro.linux.nas — аналогичный сборник инструкций для управления NAS (Network Attached Storage — компьютер, предназначенный для хранения файлов с доступом к ним по сети; не путать с NASA, которая запускает астронавтов и прочий полезный груз в космос) на основе OMV (OpenMediaVault — свободное ПО для NAS на основе Debian).

===== Условности bystro.linux =====

Все инструкции расположены по разделам:

* [[Common]]
* [[Debian]]
* [[Manjaro]]

Навигация по документу выстроена через индекс страниц — https://testitquickly.com/bystro.linux/index.html (она же полная иерархя инструкций в проекте). В Zim невозможно упорядочить/отсортировать страницы по своему хотению в индексе, а именовать файлы порядковыми цифрами — безумно. Поэтому в каждый головной файл каждого раздела должны быть добавлены ссылки на все страницы/подстраницы.

Для отображения в браузере все файлы были экспортированы из Zim в формат html нативным способом. Местами возможны потери картинок или чехарда с линками. Шаблон удобен для просмотра с большого экрана.

Тэги в начале каждого документа в версии html смотрятся как излишнее декорирование, но внутри Zim они автоматически учитываются и собираются в отдельном разделе и здорово помогают в поиске.

===== Открыть bystro.linux в Zim =====

==== Установка zim ====

* [[Debian:Установка ПО:zim]]
* [[Manjaro:Установка ПО:zim]]

==== Снять файлы ====

1. Выкачать все файлы из https://github.com/testitquickly/bystro.linux/ Большей частью там plain txt, есть немного изображений.
2. Открыть в Zim файл „notebook.zim”.

Также можно добавить новый блокнот во внутренний список блокнотов Zim:

''Открыть Zim''
''> File''
''> Open Another Notebook (Ctrl+O)''
''> Add''
''> [Browse]''
''> Указать каталог с этими файлами.''

Подхватится файл notebook.zim, блокнот откроется.
