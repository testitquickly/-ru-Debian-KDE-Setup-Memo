====== Kile ======

[ @debian @install @kile ]

===== Про нюансы установки =====

Kile при установке тащит из репозиториев Debian тамошний LaTeX. В обычных условиях этого достаточно. Я хочу установить его без texlive.

В принципе это можно сделать через dummy-пакет deb от [[+Equivs]], чтобы установщику Kile «показалось», что LaTeX из репозитория уже установлен в системе. Во времена медленного интернета это было разумным решением.

Сегодня проще 
* установить Kile с параметром «игнорировать предлагаемые зависимости», 
* проигнорировать базовые пакеты TeX (там файлов на сто мегабайт)
* [[Debian:Install Software:LaTeX|установить LaTeX из iso]] в ''/opt'' со своим блэкджеком, 
* и настроить обращение QuickBuild из Kile к файлам LaTeX из iso.

===== Установить Kile =====

…с минимальными зависимостями:

''sudo apt-get install --no-install-recommends kile kile-l10n''

Отдельно нужен ''kile-l10n'' — локализация интерфейса Kile.

==== Проверить работоспособность Kile ====

''which tex''

Ответ «''/usr/bin/tex''» означает, что в системе установлен LaTeX из репозиториев Debian. Ок, пусть лежит. Отсутствие ответа означает, что LaTeX в системе нет.

'''
which kile
'''

Ответ «''/usr/bin/kile''» нас полностью устраивает.

1. Запустить Kile.
	По-умолчанию запустится System Check — будут проверены нужные программы в ''/usr/bin''. Если нет, то пройти в ''Settings > System Check'' и посмотреть, что он скажет.	
	Может насторожить алерт на отсутствие 'Acroread', но это всего лишь уведомление о том, что не установлен Acrobat Reader, который и так не нужен (есть же 'Evince' или 'Okular'). 
2. Поставить галочки для 'Use the modern configuration for the PDFLaTeX, LuaLaTeX and XeLaTeX tools'.
3. [Finish]

Также при установке Kile может проснуться гномовский iBus, если он ранее был установлен и вроде бы подавлен — [[HowTo:Сделать:Подавить iBus]]

Далее см. установку [[LaTeX]] из iso.
